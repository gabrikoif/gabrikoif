# Gaby

I'm a self-taught developer who's been building and studying independently for a while now. Most of what I know I didn't learn in a classroom — I picked it up by going deep on things that interested me, building projects from scratch, and not stopping until I actually understood what was going on under the hood.

My work sits at the intersection of software and math. I'm comfortable with systems-level programming in **C and C++**, scripting and algorithm work in **Python**, and I've independently covered college-level topics in **linear algebra**, calculus, and physics — applying them directly in code rather than just on paper.

I'm at home in a **Linux environment**, and I work comfortably in team settings using **Git** — branching, merging, pull requests, the whole workflow.

### What drives how I work

I don't reach for libraries or abstractions until I understand what they're replacing. When I build something, I want to know why every piece is there — which means I've spent a lot of time deriving things from scratch, whether that's physics equations, data structure implementations, or build systems.

That approach takes longer upfront, but it means I can actually debug, adapt, and extend things when they break or need to change.
